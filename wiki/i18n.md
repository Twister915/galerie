## Configuration

Enable languages in `site.toml`:

**All 20 languages:**
```toml
all_languages = true
```

**Select specific languages:**
```toml
[[languages]]
code = "en"

[[languages]]
code = "zh_CN"

[[languages]]
code = "ja"
```

See [Site Configuration](site-config.md#all_languages-optional) for more options.

## How It Works

- **Client-side switching**: All translations are embedded as JSON. Language switching happens instantly via JavaScript without page reloads.
- **Browser detection**: On first visit, the user's browser language preferences are checked and the best match is auto-selected.
- **Persistence**: User's language choice is saved in `localStorage` and persists across sessions.
- **Fallback**: If a translation key is missing, English is used as fallback.

## What Gets Translated

- **UI elements**: Navigation labels, section headers, field names, action buttons
- **Country names**: ~50 common countries are translated in all languages
- **Footer branding**: "Built with galerie" adapts to each language's word order

## Theme Integration

Both built-in themes (`basic` and `fancy`) support i18n. For the `fancy` theme:
- A dropdown language picker appears in the header when multiple languages are configured
- The info drawer metadata labels are translated
- Country names in GPS location data are localized

## Custom Themes

### HTML: data-i18n Attribute

Use the `data-i18n` attribute on elements for static text:

```html
<span data-i18n="nav.previous">Previous</span>
<button data-i18n="action.download">Download</button>
```

The English text is the fallback if translations aren't loaded.

### JavaScript: t() Function

Use the global `t()` function in JavaScript:

```javascript
var label = t('field.camera');  // Returns "Camera" or translated equivalent
var country = t('country.' + photo.gps.countryCode);  // "Japan" or "日本"
```

### Countries

Country names use ISO 3166-1 alpha-2 codes:

```javascript
t('country.US')  // "United States" or translated
t('country.JP')  // "Japan" or "日本"
t('country.FR')  // "France" or "فرنسا" (Arabic)
```

~50 common countries are included. Unknown codes return the code itself.

## RTL Support

Arabic (`ar`) and Hebrew (`he`) are right-to-left languages. The built-in themes handle RTL layout automatically when these languages are selected.

## Word Order Differences

Some languages (Japanese, Chinese, Hindi, Korean) place verbs after nouns. The footer uses a prefix/suffix pattern:

- English: "Built with galerie" (`footer.built_with` + "galerie")
- Japanese: "galerie で構築" ("galerie" + `footer.built_with_suffix`)

This pattern is available for custom themes that need similar flexibility.
